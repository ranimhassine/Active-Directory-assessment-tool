Prioritized Recommendations

The following 11 recommendations were generated based on the findings detected in this assessment. They are ordered by urgency - address Immediate items first to close the highest-risk exposure windows.

⚠ Immediate Actions 2 recommendations
1
Remove Unauthorized DCSync Rights
P1 - Critical Effort: 2-4 hours
What & Why
Non-DC accounts holding Replicating Directory Changes All rights can extract all domain password hashes, leading to instant full domain compromise.
How To Fix
1. Run: (Get-ACL 'AD:\DC=domain,DC=local').Access | Where ObjectType -eq '1131f6ad...' to identify holders.
2. Use ADSI Edit or PowerShell to remove the ACE from unauthorized principals.
3. Monitor domain root ACL changes with audit policy.
Recommended Tools: ADSI Edit, PowerShell, BloodHound (to verify)
2
Disable Print Spooler on All Domain Controllers
P1 - Critical Effort: 30 minutes
What & Why
Print Spooler on DCs enables PrintNightmare (CVE-2021-34527) and SpoolSample - both allow SYSTEM-level code execution and credential coercion.
How To Fix
1. Run on each DC: Stop-Service Spooler -Force; Set-Service Spooler -StartupType Disabled
2. Deploy via GPO: Computer > Windows Settings > System Services > Print Spooler = Disabled
3. Verify: Get-Service -ComputerName -Name Spooler
Recommended Tools: PowerShell, Group Policy Management Console (GPMC)
▶ Short-Term Actions 3 recommendations
3
Enforce Strong Domain Password Policy
P2 - High Effort: 2-4 hours
What & Why
A minimum password length below 12 characters significantly increases risk of successful brute-force and credential spray attacks.
How To Fix
1. Open GPMC > Default Domain Policy > Computer Config > Windows Settings > Security Settings > Account Policies
2. Set: Minimum password length = 14, Complexity = Enabled, History = 24, Max age = 90 days
3. Consider Microsoft LAPS for local admin passwords.
4. Use Fine-Grained Password Policies (PSO) for privileged accounts requiring stricter settings.
Recommended Tools: GPMC, PowerShell: Set-ADDefaultDomainPasswordPolicy
4
Enable Account Lockout Policy
P2 - High Effort: 1-2 hours
What & Why
Without an account lockout threshold, attackers can attempt unlimited password guesses against any account, enabling password spray and brute-force attacks.
How To Fix
1. Default Domain Policy > Account Lockout Policy:
- Threshold: 5-10 invalid attempts
- Duration: 15-30 minutes
- Reset counter: 15 minutes
2. For privileged accounts, use Fine-Grained PSO with stricter lockout (3 attempts).
3. Monitor: Event ID 4740 (account lockout).
Recommended Tools: GPMC, PowerShell: Set-ADDefaultDomainPasswordPolicy
5
Mark Privileged Accounts as Sensitive and Remove Service Accounts from Admin Groups
P2 - High Effort: 4-8 hours
What & Why
Privileged accounts not marked sensitive can have credentials delegated through Kerberos. Service accounts in admin groups are frequently exploited for privilege escalation.
How To Fix
1. Mark sensitive: Set-ADUser -AccountNotDelegated $true
2. Remove service accounts: Get-ADGroupMember 'Domain Admins' | where name -match 'svc' | remove..
3. Create dedicated service account OUs with minimal permissions
4. Use gMSA or LAPS where applicable.
Recommended Tools: PowerShell AD module, ADUC
● Medium-Term Actions 3 recommendations
6
Clean Up Stale and Unused Accounts
P3 - Medium Effort: 1-2 days initial cleanup, ongoing process
What & Why
Stale and never-used accounts expand the attack surface, are frequently overlooked in access reviews, and are prime targets for credential stuffing.
How To Fix
1. Export stale accounts: Get-ADUser -Filter {LastLogonDate -lt (Get-Date).AddDays(-90) -and Enabled -eq $true}
2. Stage: Move to a 'Quarantine' OU, disable accounts (do not delete immediately)
3. After 30-day hold with no complaints, delete
4. Implement automated deprovisioning via ITSM/IDM integration
5. Set up recurring quarterly access reviews.
Recommended Tools: PowerShell, Microsoft Identity Manager, Entra ID Governance
7
Set Machine Account Quota to Zero
P3 - Medium Effort: 1 hour
What & Why
A non-zero ms-DS-MachineAccountQuota lets any authenticated user create computer accounts, enabling RBCD and persistence attacks.
How To Fix
1. Set to 0: Set-ADDomain -Identity -Replace @{'ms-DS-MachineAccountQuota'='0'}
2. Delegate computer account creation to specific admin accounts or OUs only
3. Use a service account with restricted rights for workstation joins (e.g., via SCCM/Intune).
Recommended Tools: PowerShell AD module, ADSI Edit
8
Deploy Accounts with Password Never Expires Remediation Plan
P3 - Medium Effort: 1-2 weeks
What & Why
Accounts with non-expiring passwords are high-value targets since any compromised credential remains valid indefinitely.
How To Fix
1. For standard users: Remove PasswordNeverExpires flag and enforce policy
2. For service accounts: Migrate to gMSA (auto-rotating passwords)
3. For break-glass accounts: Store in PAM vault with check-out/check-in workflow
4. Create a Fine-Grained PSO for service accounts with extended expiry but not unlimited.
Recommended Tools: PowerShell, gMSA, CyberArk/BeyondTrust PAM
◆ Long-Term Actions 3 recommendations
9
Implement Active Directory Tiering Model
P4 - Strategic Effort: 2-4 weeks
What & Why
Without tiering, privileged credentials from Tier 0 (DCs, AD) are exposed on lower-trust systems enabling credential theft and lateral movement.
How To Fix
1. Define three tiers: Tier 0 (AD/DC), Tier 1 (Servers), Tier 2 (Workstations)
2. Create separate admin accounts per tier (e.g., admin.t0@domain, admin.t1@domain)
3. Enforce via Authentication Policy Silos (Windows Server 2012 R2+)
4. Deploy Privileged Access Workstations (PAWs) for Tier 0 administration
5. Use GPO to restrict logon rights per tier (Deny log on locally, Deny log on through RD Services)
Recommended Tools: GPMC, Authentication Policy Silos, PAW deployment guide (Microsoft)
10
Conduct GPO Baseline Review and Security Hardening
P4 - Strategic Effort: 1-2 weeks
What & Why
Unlinked and unreviewed GPOs create configuration drift. Missing security baselines leave credential protection, NTLM hardening, and audit policies ungoverned.
How To Fix
1. Run Microsoft Security Compliance Toolkit - download baselines for your OS versions
2. Use Policy Analyzer to compare current GPOs against baseline
3. Remove/archive all unlinked GPOs after review
4. Apply CIS or Microsoft baseline GPOs in audit mode first, then enforce
5. Implement: LSA Protection, Credential Guard, WDigest disabled, AppLocker/WDAC.
Recommended Tools: Microsoft Security Compliance Toolkit, CIS Benchmarks, GPMC, Policy Analyzer
11
Implement Continuous AD Security Monitoring
P4 - Strategic Effort: 2-4 weeks
What & Why
Point-in-time assessments miss changes that occur after the scan. Continuous monitoring detects privilege escalation, persistence, and lateral movement in real time.
How To Fix
1. Enable Advanced Audit Policy (GPO) for: Account Logon, Account Management, DS Access, Privilege Use, Logon/Logoff
2. Forward logs to SIEM (Sentinel, Splunk, QRadar)
3. Create alerts for: DA group changes, DCSync activity, AdminCount changes, Kerberoast spikes (4769), Pass-the-Hash (4768+4769 anomalies)
4. Consider dedicated AD monitoring: Microsoft Defender for Identity (MDI), Semperis DSP, or Netwrix Auditor.
Recommended Tools: Microsoft Defender for Identity, Azure Sentinel, Netwrix, Semperis DSP

Impact: Allows attackers to create computer accounts for RBCD, Kerberoasting, and persistence attacks with only standard user credentials.

Remediation: Set ms-DS-MachineAccountQuota to 0. Delegate computer join rights to specific accounts or OUs only.

Technical Details: Current MAQ: 10
